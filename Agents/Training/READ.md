# 📚 Training Knowledge Base

The files contained within this directory serve as the structured knowledge sources used to train and improve TechPixel Agent's understanding of Bloxd.io.

Each document focuses on a specific area of the game and provides reference material that can be used for answering questions, generating content, validating information, and improving response quality.

---

## Purpose

The Training Knowledge Base exists to help TechPixel provide responses that are:

- More accurate
- More Bloxd-specific
- Better grounded in documented mechanics
- Consistent across conversations
- Less likely to hallucinate information

These files supplement the Agent's general reasoning capabilities with focused Bloxd knowledge.

---

## Training Modules

### `Arsenal.pdf`
Contains information related to weapons, combat systems, damage mechanics, equipment behavior, and offensive gameplay strategies.

### `Coding.pdf`
Includes WorldCode examples, scripting references, debugging techniques, API usage patterns, and programming concepts relevant to Bloxd development.

### `Dev and Social.pdf`
Documents community practices, development workflows, BloxdHub conventions, collaboration guidelines, and social/community-related knowledge.

### `Enchantments.pdf`
Provides reference material covering enchantments, effects, compatibility, balancing considerations, and gameplay interactions.

### `Gamemodes.pdf`
Contains information about official and custom game modes, mechanics, objectives, progression systems, and implementation ideas.

### `Mobs.pdf`
Documents entities, mob behaviors, abilities, interactions, spawning mechanics, and related gameplay systems.

### `Placeable Blocks.pdf`
Includes details regarding blocks, placement behavior, interactions, environmental systems, and building mechanics.

### `Settings.pdf`
Provides information regarding game settings, configuration options, customization systems, and user preferences.

## Validation Pipeline

Submitted training materials are reviewed by the Training Note Checker before being accepted into the knowledge base.

The validation process helps ensure that documentation remains:

- Relevant
- High quality
- Bloxd-focused
- Consistent with existing knowledge
